TypeScript 的使用

ts 网上都有完整的教程，我这里就不多说了。只说一些，我的项目里 用的的一些实用的东西。ts网上的文档：点击前往

基础类型

boolean：布尔类型
number：TypeScript所有的数值类型采用浮点型计数
string：字符串类型
Array：数组
Enum：枚举
any：任何类型，可以用来跳过TypeScript的编译时类型的检查
void：表明函数无返回值
null 空值
undefined 未定义

修饰符

public  a = 1; //公共的
private b = 2; //私有的
protected c = 3; //可继承的

    var a = 1;//js 的写法 ts 也可以，但是不推荐，推荐用let

    const c = 3;    //常量申请  不能写在类里边 可以写在 方法里。

类型别名跟接口很像 但是不能继承 它也不会生成js代码 它只起一个规范作用

两个方法 fun1 和fun2 都是传入一个 uiData的Object类型的数据，但是如果没有 UIData 类型别名的约束，就很有可能传错参数。 里边的 加一个 ?号 表示可有可无的 参数。 一些简单的类型 不用再写一个类型别名，可以直接 :号后边写出来

抽象类 抽象方法 泛型类 泛型约束

  abstract class BaseCtrl<T extends fairygui.GComponent> {
    /**控制的界面 */
    private view_: T
    /**
     * 界面初始化完成的时候调用
     */
    public initView(_view: T): void {
        this.view_ = _view;
        this.init();
    }
    /**获取界面 */
    public get view(): T {
        return this.view_;
    }
    /**关闭界面 */
    public closeView(okFun?: Function, okObj?: any): void {
        if (okFun && okFun.bind) {
            okFun.bind(okObj);
        }
        else {
            okFun = null;
        }
        App.UIManager.closeID(this.uiData.id, okFun);
    }
    /**
     * 初始化完成
     */
    abstract init(): void;
    /**
     * 界面打开的时候调用
     */
    abstract open(...param: any[]): void;
    /**
     * 界面关闭的时候调用
     */
    abstract close(): void;
    /**
     * 界面销毁的时候调用
     */
    abstract destroy(): void;
    /**
     * 获取UI数据
     */
    abstract get uiData(): UIData;

}

索引类型（Index types）

class Class1 {
    public a:number;
    public b:number;
    public getValue<U extends keyof Class1>(k: U): any {
        return this[k];
    }
    public setValue<U extends keyof Class1>(k: U, value: any): void {
        this[k] = value;
    }
}

外部调用的时候只能用 Class1的属性名字做参数。否则 编译报错

方法参数

 function fun(id:string,name?:string,sex:string="男",...hobbys:string[]):void

id是必须参数。 name是可选参数 后面加 ?号。 sex也是可选参数。有默认值。 hobbys是剩余参数 如果你并不知道有几个参数传进来，可以用这个表示。
